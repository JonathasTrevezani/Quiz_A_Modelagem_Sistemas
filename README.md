## 📖 Sobre o Projeto

Este é um **quiz interativo** desenvolvido para auxiliar no estudo da disciplina **Análise e Modelagem de Sistemas**. O projeto foi criado com base no material de estudo da **Uninter** (aulas 1 a 6), oferecendo uma ferramenta prática para autoavaliação e revisão dos conceitos fundamentais de análise de sistemas, modelagem de processos e engenharia de requisitos.

## 🎯 Finalidade

* **Uso Acadêmico:** complemento aos estudos da disciplina, permitindo que alunos testem seus conhecimentos de forma interativa.
* **Uso Pessoal:** ferramenta para revisão de conteúdos antes de provas e avaliações.
* **Recurso Didático:** material de apoio para professores e tutores da disciplina.

## ✨ Funcionalidades

* ✅ Questões objetivas baseadas nos conteúdos de Análise e Modelagem de Sistemas
* ✅ Questões discursivas para aprofundamento teórico
* ✅ Navegação intuitiva por números
* ✅ Feedback imediato com resolução passo a passo
* ✅ Barra de progresso

* ✅ Design responsivo

## 📚 Conteúdo Abordado

| Ordem | Tema Principal |
| ----- | -------------- |
| 1 | Processos de Negócio e BPM (Business Process Management) |
| 2 | BPMN (Business Process Model and Notation) |
| 3 | Mapeamento AS IS e TO BE |
| 4 | Análise de Sistemas e Análise Estruturada |
| 5 | Requisitos Funcionais e Não Funcionais |
| 6 | Documento de Especificação de Requisitos |
| 7 | Casos de Uso e Fluxos (Principal e Alternativo) |
| 8 | UML (Unified Modeling Language) – Visão Geral |
| 9 | Diagrama de Classes – Conceitos e Relacionamentos |
| 10 | Orientação a Objetos (Abstração, Encapsulamento, Herança) |
| 11 | Diagrama de Sequência e Diagrama de Colaboração |
| 12 | Diagrama de Estado – Eventos e Transições |
| 13 | Diagrama de Componentes |
| 14 | Diagrama de Fluxo de Dados (DFD) |
| 15 | Análise de Requisitos e o Papel do Analista |

## 🧠 Temas Discursivos

1. **Requisitos Funcionais e Não Funcionais** – definições, diferenças e exemplos práticos
2. **UML: Visão Estrutural e Comportamental** – diferenças entre os diagramas estáticos e dinâmicos
3. **Diagrama de Classes: Conceitos e Relacionamentos** – classes, atributos, métodos, herança, encapsulamento e abstração
4. **BPM, BPMN e Mapeamento de Processos (AS IS e TO BE)** – conceitos e aplicações na modelagem de negócios
5. **Análise de Requisitos e Documento de Especificação** – importância, técnicas de levantamento e estrutura do DER

## 🚀 Como Usar



O quiz está disponível através do GitHub Pages:

🔗 **[[### Acesse Online](https://jonathastrevezani.github.io/Quiz_A_Modelagem_Sistemas/)]**

### Executar Localmente

1. Clone o repositório:

```bash
git clone https://github.com/JonathasTrevezani/Quiz_A_Modelagem_Sistemas.git

2. Entre na pasta do projeto:

```bash
cd Quiz_Design_Interacao
```

3. Abra o arquivo `index.html` em seu navegador.

## 📁 Estrutura do Projeto

```text
Quiz-Analise-Modelagem/
├── index.html      # Estrutura principal do quiz
├── style.css       # Estilos e layout responsivo
├── script.js       # Lógica do quiz

🛠️ Tecnologias Utilizadas
HTML5 – Estrutura da aplicação

CSS3 – Estilização e responsividade

JavaScript – Lógica do quiz e manipulação do DOM

📊 Sistema de Pontuação
Cada questão vale aproximadamente 6,7 pontos.

São 15 questões, totalizando 100,5 pontos.

O resultado é calculado automaticamente ao final do quiz.

## 🔧 Personalização

Para adicionar ou modificar questões, edite o array `questions` no arquivo `script.js`:

```javascript
{
  text: "Enunciado da questão",
  alternatives: [
    "Alternativa A",
    "Alternativa B",
    "Alternativa C",
    "Alternativa D"
  ],
  correct: "B",
  explanation: "Resolução detalhada da questão..."
}
```
